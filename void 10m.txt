Количество неудачных попыток в Nginx определяется параметром `client_max_body_size` в конфигурационном файле `nginx.conf`. Для установки значения параметра нужно добавить или изменить его в блоке `http`.

Пример:

```
http {
  client_max_body_size 10m;
  ...
}
```

В данном примере установлено максимальное количество данных, которое может принять сервер, равное 10 мегабайтам. Если клиент отправляет на сервер данные размером больше, чем указано в параметре, то соединение будет закрыто с ошибкой.

Обратите внимание, что параметр `client_max_body_size` не относится к неудачным попыткам авторизации на сервере. Для этого нужно использовать модуль `http_auth_request`. Этот модуль позволяет выполнять запрос к другому серверу для проверки пользовательского имени и пароля. Если запрос вернет ошибку, то соединение будет закрыто с ошибкой.
